Getting Started

Supported Devices

In order to use MyoKit, you need to have a device that supports BLE. They are as follows:

  • iPhone 4S or later
  • iPad 3rd gen or later
  • iPod Touch 5th gen or later
  • iPad mini

Adding MyoKit to a Project

  1. Add the MyoKit.embeddedframework folder to your Xcode project.
  2. While still in 'Build Settings', search for 'Other Linker Flags' and add -ObjC.
  3. In 'Build Phases', add MyoKit.framework, CoreBluetooth.framework, and SystemConfiguration.framework to 'Link Binaries With Libraries'.

Instantiating the Hub

In your AppDelegate.m, add #import <MyoKit/MyoKit.h>.

In your AppDelegate's application:didFinishLaunchingWithOptions: method, add the following line: 

- (BOOL)application:(UIApplication *)application didFinishLaunchingWithOptions:(NSDictionary *)launchOptions {

    /**
    Default code.


    [TLMHub sharedHub];
    return YES;
}

Managing Myos

The Myo iOS SDK comes with a built in view controller for connecting to a particular Myo called the TLMSettingsViewController. There are several ways to create the view controller, but when it's being presented to the user, it must be in a UINavigationController. Here's several ways to show users the Myo settings VC: 

// Pushing it on an existing navigation controller
- (void)pushMyoSettings {
    TLMSettingsViewController *settings = [[TLMSettingsViewController alloc] init];

    [self.navigationController pushViewController:settings animated:YES];
}

// Presenting modally
- (void)modalPresentMyoSettings {
    UINavigationController *settings = [TLMSettingsViewController settingsInNavigationController];

    [self presentViewController:settings animated:YES completion:nil];
}

// Presenting in a Popover (iPad only)
- (void)popoverPresentMyosettings {
    // Make sure to hold a strong reference to your popover so it doesn't get dealloc'ed when it's presented.
    self.popover = [TLMSettingsViewController settingsInPopoverController];

    [self.popover presentPopoverFromBarButtonItem:settingsButton 
                         permittedArrowDirections:UIPopoverArrowDirectionAny 
                                         animated:YES];
}

Alternatively, you can connect to a Myo programatically using the -(void)pairWithAny method, or the - (void)pairWithAdjacent method. 

//Connecting to a Myo with pairWithAny
- (void)connectToMyo {
    [[TLMHub sharedHub] pairWithAny];
}

//Connecting to a Myo with pairWithAdjacent
- (void)beginAdjacentConnection {
    [[TLMHub sharedHub] pairWithAdjacent];
}

//Make sure to turn it off if you don't want to connect to any other Myos
- (void)endAdjacentConnection {
    [[TLMHub sharedHub] stopPairWithAdjacent];
}

Training a Myo

In order to train a Myo, use the presentTrainerFromViewController: method on TLMMyo. The view controller passed in will be used to present the training view controller modally. It will dismiss itself when it has completed. Training data will be stored in user defaults, and loaded whenever the Myo reconnects. The Myo will have to be retrained any time the Myo is removed from your forearm. Here is an example of how to present the training view controller: 

- (void)presentTrainer {
    TLMMyo *device = [[[TLMHub sharedHub] myoDevices] firstObject];
    if(device) {
        [device presentTrainerFromViewController:self];
    }
}

Receiving Pose and Orientation Events

Now that the SDK is integrated to your project and the device is trained, it's time to start receiving data. New data notifications are received through the NSNotificationCenter built into iOS.
